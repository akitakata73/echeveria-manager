# エケベリア管理 PWA v27 Manual Recovery

## v27の目的

v26 Stableで発生した「過去キーの自動採用による巻き戻り」を止めるための修正版です。

## 原因

v26では、起動のたびに旧保存キーを全探索し、株数・履歴数・写真数などのスコアが最大のデータを自動採用していました。

そのため、v25/v26で新規入力していても、v21のデータ量が多ければ、毎回v21が採用され、現在データが巻き戻る可能性がありました。

## v27の変更

- 通常起動では固定キー `echeveria_pwa_data` を最優先
- 固定キーが存在する場合、旧キー探索で勝手に上書きしない
- 旧キー探索は診断表示用として維持
- 設定画面で各キーの株数・履歴数・写真数・最新日時を表示
- 必要な場合だけ「このデータを採用」ボタンで手動復旧
- 読込元、保存キー、株数、履歴数、写真数を設定画面に表示

## 重要

v27では自動でv21へ巻き戻る動作を止めています。
もし15時頃の入力が残っているキーが診断に表示されていれば、そのキーの「このデータを採用」を押してください。
