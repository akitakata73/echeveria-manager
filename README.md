# エケベリア管理 PWA v40 Safe Commit

## v40の目的

v39の写真切り離し方針を維持しつつ、タスクキル後に入力データが一定地点まで戻る問題を防ぐため、保存確定処理を修正した版です。

## 主な変更点

- 本番保存を最優先に変更
- previous退避を本番保存の後に変更
- previous退避は、旧データを軽量化して保存
- previous退避に失敗しても、本番保存は成功扱い
- 本番保存後に読み戻し検証を実施
- 保存失敗時は本番データを上書きしない
- アプリ本体更新中は pagehide / visibilitychange の自動保存を抑制
- アプリ本体更新ボタンは、保存・緊急バックアップ成功時のみ実行

## 保存順序

1. 写真なし軽量JSONを生成
2. tmpキーに保存
3. tmpキーを読み戻して検証
4. 本番キーへ保存
5. 本番キーを読み戻して検証
6. 旧本番データを軽量化してpreviousへ退避
7. previous退避に失敗しても本番保存は維持

## 維持している固定ルール

- 保存キー：echeveria_pwa_data
- 直前データ：echeveria_pwa_previous_data
- 写真本体：IndexedDB
- 旧キー自動採用なし
- 保存データ診断・復旧候補欄なし
- localStorage.clear() 未使用
